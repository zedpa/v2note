# 念念有路（v2note）PC前端设计方案

> 版本：1.1 · 2026年3月
> 
> 品牌名：念念有路 · v2note
> 吉祥物：路路（Lùlù）— 一只温暖、安静、指路不催路的小鹿
> Slogan：你说的每一句话，都会变为行动
> 
> 核心理念：它首先是一个极致纯粹的日记工具。然后，如果你愿意，它会慢慢变成一面认知的镜子。

---

## 一、设计哲学

### 1.1 三条宪法

**输入是神圣的。** 任何时候打开 v2note，都是一张白纸。用户的文字、语音、图片、文件——这些是第一公民，所有其他功能都是为它们服务的。

**结构是涌现的。** 用户不创建分类、不设标签体系、不搭建层级。他只管记录。结构从他的记录中自动生长——主题、目标、项目，一层一层往上涌现。用户看到的是"我的世界自己长出了形状"。

**AI 不说话，除非被问。** 系统的智能体现在沉默的准确（打标签、做关联）和定时的报告（每日回顾），不体现在随时随地的弹窗和提醒。

### 1.2 不暴露的概念

以下技术概念永远不出现在用户界面中：

| 系统概念 | 用户看到的 |
|---------|-----------|
| Strike / 原子 | （不可见，后台处理） |
| Link / Bond | "关联"、"相关" |
| Cluster | "主题" |
| 极性（Perceive/Judge/Realize/Intend/Feel） | （不可见，驱动后台逻辑） |
| Strength / 置信度 | 关联度百分比条，或语气差异 |
| 涌现 | "路路发现"、"路路注意到" |

用户从头到尾接触的概念只有：日记、主题、目标、项目、关联、行动、路路的发现。

### 1.3 AI 三层参与度

| 层级 | 名称 | AI 做什么 | 用户感知 | AI 姿态 |
|-----|------|----------|---------|---------|
| L1 | 工具 | 打标签、做链接、聚类、涌现结构 | 日记自动归到主题下，地图自动生长 | 沉默的仆人——做完就走，不解释 |
| L2 | 报告 | 每天两次定时报告（晨间+晚间） | 每日回顾中看到行动建议和洞察 | 安静的秘书——递交报告，等你翻阅 |
| L3 | 参谋 | 回答主动提问，深度分析，角色扮演 | 我问它才说话，引用我的日记做论据 | 被请教的顾问——叫它才来 |

### 1.4 确定性骨架 + 概率性血肉

| 确定性骨架（无 AI 也能用） | 概率性血肉（AI 增强） |
|--------------------------|---------------------|
| 日记的原文保存 | 自动主题标签 |
| 时间戳 | 日记间关联发现 |
| 全文搜索 | 目标/项目涌现 |
| 用户手动标签（@ #） | 认知地图生成 |
| 用户手动创建的目录/目标 | 目标叙事 |
| 文件/图片附件 | 每日回顾洞察 |

降级策略：AI 不可用时退化为纯日记工具。时间线正常显示，搜索正常工作。地图冻结在最后状态，标签停止更新，每日回顾只显示日记摘要。菜单栏状态点变灰。

## 二、视觉语言

### 2.1 配色体系

取自路路的自然生态——树皮、鹿毛、苔藓、晨光。温暖但不甜腻，沉稳但不沉闷。

核心色：

| 名称 | 色值 | 用途 |
|------|------|------|
| 树皮 Bark | #3D3228 | 品牌主色、标题、按钮、当前导航高亮 |
| 鹿毛 Deer | #C8845C | 品牌辅色、Logo、强调、品牌色背景 |
| 鹿角 Antler | #A06B42 | 活跃状态、可点击链接 |
| 暖奶 Cream | #FAF6F0 | 页面背景 |
| 暖沙 Sand | #F0E8DB | 侧边栏、面板背景、代码区底色 |
| 边界 Border | #E0D5C8 | 所有分割线和边框 |

文字色阶：

| 用途 | 色值 |
|------|------|
| 主文字 | #2C2520 |
| 次文字 | #6B5E52 |
| 弱文字 | #9B8E82 |
| 隐文字 | #C4B8AC |

语义色（极度克制）：

| 名称 | 色值 | 使用场景 |
|------|------|---------|
| 森林 Forest | #5C7A5E | 已完成、正面状态 |
| 天空 Sky | #7BA3C4 | 关联度、可跳转、信息类 |
| 晨光 Dawn | #E8A87C | 路路的发现、提醒 |
| 枫红 Maple | #C45C5C | 跳过标记、风险、录音状态 |

### 2.2 字体

| 场景 | 字体 | 字号 |
|------|------|------|
| 品牌标题 | 'Noto Serif SC', serif | 按场景定 |
| 写作面板 | 'SF Mono', 'Fira Code', monospace | 15px / line-height: 2 |
| 界面正文 | 'Noto Sans SC', sans-serif | 14px / line-height: 1.6 |
| 标签/辅助 | 同上 | 11-12px |

### 2.3 路路在产品中的出现

| 场景 | 路路的存在方式 |
|------|--------------|
| 菜单栏 Logo | 路路头像 + "念念有路" |
| 每日回顾 | "路路发现……" |
| 降级状态 | 路路头像变灰 |
| 冷启动/空状态 | 路路完整形象 + 引导文案 |
| 参谋对话 | 对话头像 |
| 其他场景 | 不出现 |

### 2.4 设计规则

- 不使用紫色渐变、不使用冷灰色调
- 圆角不超过 12px，阴影不超过 shadow-sm
- 背景色不超过两层暖灰嵌套（Cream → Sand）
- 图标优先使用 emoji
- 动画时长不超过 200ms，过渡 ease-out
- AI 输出用路路口吻（温暖、不催促、不评判）

## 三、全局架构

### 3.1 场景模型

```
v2note
├── 场景A：写作（默认，纯输入面板）
├── 场景B：时间线（三栏：结构 + 日记流 + 焦点）
├── 场景C：认知地图（全屏画布，网状图/思维导图切换）
├── 场景D：目标（卡片流：项目→目标→行动）
│
├── 常驻浮层：⚡️ 行动队列（右侧边栏）
├── 常驻浮层：📋 每日回顾（居中弹窗）
└── 嵌入能力：💬 参谋对话（内嵌在触发位置）
```

### 3.2 菜单栏

鼠标移到顶部 48px 区域滑入，离开 400ms 后淡出。

```
┌────────────────────────────────────────────────────────────────────┐
│ [🦌] 念念有路  写作  时间线  地图  目标  ···  🔍 🎙 ⚡️行动 📋回顾 ⚙️ │
└────────────────────────────────────────────────────────────────────┘
```

暖奶色底，底部 1px 边界色边框，高度 44px，backdrop-blur。

## 四、场景 A：写作

任何时候的默认场景。打开念念有路，看到的就是这里。

### 4.1 布局

全屏只有一个居中编辑区，最大宽度 680px，上下留白充足。

```
┌──────────────────────────────────────┐
│          （菜单栏 hover 显现）         │
├──────────────────────────────────────┤
│                                      │
│  3月21日 周六                         │
│  ─────────────                       │
│                                      │
│  今天和张总开会，他说原材料             │
│  涨了15%，我觉得我们应该换             │
│  供应商......|                        │
│                                      │
│                                      │
├──────────────────────────────────────┤
│  Markdown │ 3行·47字  Ctrl+Enter 提交 │  hover 显现
└──────────────────────────────────────┘
```

### 4.2 输入方式

- **打字**：Markdown 语法，等距字体，行高 2 倍
- **语音**：菜单栏 🎙 或 Ctrl+Space，录音时红色脉冲，文字自动插入编辑区
- **拖放**：文件/图片/链接拖入，虚线边框 + "松开即插入"，以 block 格式插入
- **粘贴**：大段文本以附件块形式插入（类似素材），AI 不做深度处理
- **提交**：Ctrl+Enter 发送，Ctrl+S 暂存（自动保存每 30s）

### 4.3 特殊输入

- **/ 命令**：行首 `/` 弹出命令面板（Markdown 语法提示 + 高频菜单入口 + 场景导航）
- **@ 链接**：`@` 弹出结构列表（主题、目标），选择后插入关联标记
- **# 标签**：`#` 弹出标签列表（已有 + 新建），用户手动标签权重高于 AI 自动标签。`#`+空格 识别为 Markdown 标题语法

### 4.4 反馈

提交后底部短暂显示："✓ 路路收到了 · 关于供应商和成本"（1.5 秒淡出）。固定时长前端动效，不等 AI。AI 标签和关联异步完成后直接更新到时间线。

## 五、场景 B：时间线

浏览和回顾日记。三栏布局。

### 5.1 布局

```
┌────────┬─────────────────────────────────┬───────────────────────┐
│        │ 筛选: 全部 | 语音 | 文字 | 图片  │                       │
│ 结构   │                                 │                       │
│ 导航   │ Zed · 09:12 🎙                  │   焦点                 │
│        │ 今天和张总开会，他说原材料         │   侧边栏               │
│ ────── │ 涨了15%......                    │                       │
│ 供应链  │ ┄ 供应链 · 成本控制              │  （点击日记后展开）     │
│  └成本  │ 💬3  🔗5条关联                   │                       │
│  └供应商│ ───────────────────             │                       │
│ v2note │ Zed · 09:35 ✏️                  │                       │
│  └UI   │ 用户要的不是功能少，是            │                       │
│  └架构  │ 理解成本低......                 │                       │
│ 个人   │ ┄ v2note 产品                    │                       │
│  └交易  │ 💬1  🔗2条关联                   │                       │
│        │ ───────────────────             │                       │
│ ────── │                                 │                       │
│ +新主题 │                                 │                       │
└────────┴─────────────────────────────────┴───────────────────────┘
```

### 5.2 左栏：结构导航（200px）

涌现出的结构层级 + 用户手动创建的结构。

- 点击主题 → 中栏筛选为该主题下的日记
- "+ 新主题" → 用户手动创建目录，语义相关历史日记自动归入
- 主题名可右键重命名/合并/删除
- 拖拽日记到主题上 → 手动链接（高权重）
- 涌现的新主题安静出现，旁有微妙"新"小点
- 结构层级最多三层

### 5.3 中栏：日记流

推特式卡片流。每条包含：头像+用户名+时间戳+输入方式图标 → 原文 → 底部主题标签（AI 自动标注）+ 💬评论数 + 🔗关联数。

筛选器：全部 | 语音 | 文字 | 图片 | 带文件 + 时间范围。

日记流中**不插入** AI 观察卡片（报告不散落原则）。

### 5.4 右栏：焦点侧边栏（320px，点击触发）

点击日记后展开：原始记录（完整+语音回放）→ 附件 → 🔗相关记录（带关联度百分比条）→ 📂所属主题/目标 → 💬"和路路聊聊这条记录"。

## 六、场景 C：认知地图

俯瞰认知全景。全屏画布，两种视图切换。

### 6.1 顶部工具栏

```
[🌐 网状图]  [🌲 思维导图]  自动布局  缩放适配  🔍搜索节点
```

### 6.2 网状图（发散视图）

回答：我的世界长什么样？各部分之间有什么隐性关联？

- 白底 + 32px 网格背景
- 节点 = 主题（白色圆角卡片，2px 彩色边框，名称+日记数量+活跃度）
- 连线 = 贝塞尔曲线，粗细代表跨主题日记频率
- 节点距离 = 语义关联度
- 活跃节点饱和+阴影明显，不活跃淡化 opacity 0.7

语义缩放（双击节点）：节点展开为内部日记卡片群，卡片间连线是 AI 通过 Strike link 计算的。

### 6.3 思维导图（结构视图）

回答：我的事情怎么组织的？层级关系是什么？

```
       ┌─ 供应商管理 ─ 12条
  ┌─ 供应链 ────┤
  │             └─ 成本控制 ─── 8条
  │
我的认知 ──┼─ v2note ────┬─ UI 设计 ──── 5条
  │              └─ 后端架构 ─── 9条
  │
  └─ 个人 ──────┬─ 量化交易 ─── 6条
                └─ 成长 ─────── 4条
```

层级：项目 → 目标 → 主题 → 日记计数。

### 6.4 右栏：节点详情（320px，点击触发）

📊概览（记录数+活跃度+密度）→ 🎯相关目标 → 📝最近记录 → 🔗关联主题（带关联度条）。

## 七、场景 D：目标

查看目标进展、叙事弧线、行动项。卡片布局。

### 7.1 三层嵌套

```
项目卡片（最外层容器）
  ├── 目标卡片（项目内的方向）
  │    ├── ☐ 行动项
  │    ├── ☐ 行动项（跳过3次）
  │    └── ☑️ 已完成
  └── 目标卡片
       └── ☐ 行动项
```

项目卡片：浅灰底，圆角 12px。目标卡片：白底，圆角 8px，底部微型健康度指示器。

未归属目标：涌现出但未归入项目的目标，用户可拖入。

### 7.2 右栏详情（360px）

点击目标 → 🩺健康度（方向/资源/路径/驱动力四项进度条）→ 📖认知叙事（起点→转折→冲突→悬念，引用日记原文）→ 📝相关记录 → 💬"深入讨论"。

点击行动 → 来源日记 + 执行历史（跳过次数）+ 前置依赖 + 💬"聊聊为什么没做"。

### 7.3 涌现逻辑

日记 → Strike → 意图类 Strike 聚合 → 涌现"目标" → 多目标强关联 → 涌现"项目"。涌现的结构安静出现在"未归属目标"区域，和手动创建的**视觉上完全平等**。

## 九、浮层：每日回顾

菜单栏 📋。有新报告时显示鹿毛色小圆点。居中弹窗 620px。

### 晨间（☀️）

今日行动线（优先级排列）+ 系统洞察（偏差/模式/共振，高置信度陈述句，低置信度疑问句）。

### 晚间（🌙）

今日统计（记录数、主题分布、对比）+ 洞察 + 今日最有价值记录 + 情绪追踪 + 反思引导（路路提出一个问题 + 💬"想聊聊吗"）。

**AI 报告层的唯一出口。所有系统洞察集中在这里。**

## 十、嵌入能力：参谋对话

用户主动发起，不独占场景，嵌入在触发位置。

| 触发位置 | 上下文 |
|---------|--------|
| 时间线焦点栏 💬 | 当前日记 |
| 目标详情栏 💬 | 当前目标叙事和数据 |
| 每日回顾 💬 | 当前洞察 |
| 写作面板 /ask | 当前编辑内容 |

## 十四、渐进式解锁

| 日记数量 | 解锁功能 |
|---------|----------|
| 0+ | 写作面板、时间线 |
| 10+ | 自动主题标签 |

解锁时不弹窗。新功能安静地在菜单栏从灰变可点击。

## 十五、附件与外部素材处理

### 内容分类

- **原声（Voice）**：用户自己写/说 → 完整认知权重
- **素材（Material）**：外部引入 → 降级权重（原声的 1/5~1/10）

### 自动识别

- 打字/语音 → 原声
- 粘贴短文本（<100字）→ 默认原声
- 粘贴长文本（≥100字）→ 弹确认，默认素材
- 拖入文件/链接 → 始终素材

### 素材规则

- 不参与 Cluster 涌现（只被吸附到已有 Cluster）
- 不参与目标涌现
- 不参与每日回顾统计
- 参谋对话中可作为论据引用
- 展示为附着在日记卡片内部的低权重块
