# v2note PC 端设计方案

> **核心理念：打开就是一张纸。混沌进来，秩序涌现。**
> 
> 输入是纯粹的——一个空白画布，支持 Markdown、斜杠命令、任意粘贴。
> 输出是规范的——时间线、待执行、认知结构，都由 AI 自动涌现。
> 所有功能通过命令可达，不单独制作展示页面。

---

## 设计哲学

1. **输入面板即主界面。** 不是三栏 Dashboard。任何时候打开都是一个可以立刻开始写的地方。
2. **结构不预展示，按需呼出。** 认知地图、Cluster、Strike 不占常驻空间，通过 Tab/命令/选中呼出。
3. **AI 不逼迫用户。** 所有洞察、矛盾、模式提醒集中在每日计划和晚间总结中，不散落在界面各处。
4. **命令是一等公民。** /命令 + 快捷键 + 按钮，三种触达方式等价。

### 画布特性

- **Markdown 实时渲染**：边写边渲染，`#` 变标题，`- [ ]` 变 checkbox，`>` 变引用
- **无边界**：不分"标题"和"正文"，直接写。一条记录 = 从第一次敲键盘到按 Ctrl+Enter 发送之间的所有内容
- **粘贴一切**：
  - 粘贴文字 → 直接插入
  - 粘贴图片/截图 → 内联显示 + 上传
  - 粘贴 URL → 自动识别，渲染为链接卡片，可选"导入网页内容"
  - 粘贴文件 → 上传 + 显示文件名
- **发送后清空**：Ctrl+Enter → 内容进入 Digest 管道 → 画布恢复空白
- **发送后反馈**：画布下方浮出一行淡灰色文字"已收录 · 关联到「供应链管理」"，3 秒后消失

### 斜杠命令

在画布中输入 `/` 弹出命令面板：

```
┌──────────────────────────────┐
│  🔍 搜索命令...               │
│                              │
│  📋 /today     今日计划       │
│  🌙 /review    晚间总结       │
│  🗺 /map       认知地图       │
│  🎯 /goals     目标看板       │
│  ⚡ /actions   待办行动       │
│  💡 /think     决策分析       │
│  📊 /stats     认知统计       │
│  🔍 /search    搜索记录       │
│  📝 /timeline  时间线         │
│  ⚙️ /settings  设置          │
│                              │
│  最近使用                     │
│  📋 /today                   │
│  🗺 /map                     │
└──────────────────────────────┘
```

每个命令打开一个**浮层/弹窗**，不是跳转页面。关闭浮层回到画布。画布始终在底下。

### Tab 键：涌现结构预览

在画布中写到一半，按 Tab → 右侧滑出一个窄面板（280px），显示当前内容关联的涌现结构：

```
┌──────────────────────────────────┬──────────────┐
│                                  │ 相关结构      │
│  今天和张总讨论了供应链的问题，     │              │
│  他说原材料涨了15%...             │ 📁 供应链管理 │
│                                  │    12条 · 活跃 │
│                                  │              │
│  |                               │ 🔗 相关认知   │
│                                  │ 👁 交期延迟   │
│                                  │ ⚖️ 换供应商   │
│                                  │              │
│                                  │ [链接到此结构] │
│                                  │              │
│                                  │ ── 其他主题 ──│
│                                  │ 📁 成本控制   │
│                                  │ 📁 团队管理   │
└──────────────────────────────────┴──────────────┘
```

- 用户写的内容实时分析（embedding 匹配），显示最相关的 Cluster 和 Strike
- "链接到此结构" → 发送时自动关联到该 Cluster
- 用户可以选择不链接（关掉面板继续写）
- 再按 Tab 或 Esc → 面板收起

### 选中呼出：上下文相关 Strike

在画布中**选中一段文字**→ 浮出一个小工具栏：

```
  ┌──────────────────────────────────┐
  │ 🔍 相关认知  💡 分析  📌 创建行动 │
  └──────────────────────────────────┘
```

- "相关认知" → 弹窗显示和选中文字语义相关的历史 Strike
- "分析" → 弹窗用决策工作台分析这段内容
- "创建行动" → 将选中文字转为 Intend Strike + 待办

### /goals — 目标看板

弹出浮窗，看板视图。

## 时间线：画布下方的涌现

画布不只是输入。发送的内容不消失，它以**时间线**的形式沉淀在画布下方。

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                        开始记录...                           │
│                                                             │
│─────────────────────────── 今天 ────────────────────────────│
│                                                             │
│  16:30                                                      │
│  和张总讨论了供应链的问题，原材料涨了15%。我觉得应该考虑       │
│  换供应商，但老王反对。让小李做成本对比，下周三前给结果。       │
│  📁 供应链管理 · 👁×3 ⚖️×1 🎯×1                             │
│                                                             │
│  14:00                                                      │
│  产品 v2.1 版本功能确认完毕，下周一开始开发。                  │
│  📁 产品开发 · 🎯×2                                         │
│                                                             │
│  09:15                                                      │
│  [图片] 白板拍照 — 供应商评估矩阵                             │
│  📁 供应链管理 · 👁×4                                        │
│                                                             │
│─────────────────────────── 昨天 ────────────────────────────│
│  ...                                                        │
└─────────────────────────────────────────────────────────────┘
```

时间线中每条记录显示：
- 时间
- 原文（Markdown 渲染）
- 自动标签：归属 Cluster + 极性摘要
- 点击展开 → 查看 Strike 拆解详情

**时间线是输入的自然沉淀，不是独立视图。** 用户向下滚动画布就能看到历史。

## 视觉风格

延续 echo 参考的暗色暖铜美学：

- **画布背景**：深色 #0a0c10（近黑，护眼）
- **文字**：暖白 #cfc9bc
- **输入光标**：暖铜 #c4783a
- **命令面板**：半透明深色 + backdrop-blur
- **浮层**：深色毛玻璃，暖铜色 accent
- **时间线标签**：极淡的暖色 tag pill
- **Cluster 节点**：铜色边框，深色填充

支持 light mode 切换（暖米色 #FFF8F0 背景），但默认 dark。

## 和手机端的统一

| 概念 | 手机端 | PC 端 |
|------|--------|-------|
| 默认态 | Level -1 纯净入口 | 空白画布 |
| 输入方式 | 录音为主，文字为辅 | 文字为主，粘贴一切 |
| 行动面板 | 上滑 Tinder 卡片 | /today 浮窗 + /actions 浮窗 |
| 认知地图 | Brain 按钮→卡片墙 | /map 浮窗→react-flow 图谱 |
| AI 提醒 | AiWindow 推送 | 集中在 /today 和 /review |
| 时间线 | 独立 timeline 视图 | 画布下方自然沉淀 |
| 穿越 | 长按切换 | Tab 面板 + 选中呼出 |

**数据层完全共享**——手机上说的话，PC 上立刻能在时间线看到。

## 实施优先级

### W1: 画布 + 发送 + 时间线
- Markdown 编辑器（可用 tiptap 或 milkdown）
- Ctrl+Enter 发送 → POST /ingest
- 发送后时间线沉淀

### W2: 斜杠命令 + 浮层框架
- / 命令面板
- 浮层容器组件（可复用）
- /today + /review 两个核心浮层

### W3: Tab 涌现结构 + 选中呼出
- Tab 右侧面板
- 选中文字工具栏
- 实时 embedding 匹配

### W4: /map 认知图谱 + /goals 目标看板
- react-flow 集成
- 看板拖拽

### W5: 粘贴一切 + 附件
- 图片粘贴 + 上传
- URL 识别 + 提取
- 文件上传
