# v2note 前端愿景设计文档

> **交互宪法：产品有两面。正面是一条线（执行），背面是一张网（思考）。用户 90% 时间看到线。**

---

## 设计原则

1. **镜子，不是教练。** 呈现数据而非判断。用户自己决定"3条记录，最后14天前"意味着什么。
2. **倾倒优先。** 默认状态是纯净入口。思考不能被打断。
3. **置信度驱动 UI。** 确定的事实用陈述语气，推断用疑问语气+确认入口。
4. **用户有主权。** 允许手动建立连接、修正归类、否定推断。
5. **执行优先。** 用户 90% 的时间需要的是"告诉我现在该做什么"，不是"让我审视我的认知世界"。

## 产品双面架构

```
          ┌──────────────┐
    上滑  │   🎙 录音入口  │  下拉
   ┌──────┤   （中间层）   ├──────┐
   │      └──────────────┘      │
   ▼                            ▼
┌──────────────┐    ┌──────────────┐
│  行动面板     │    │  认知地图     │
│  （执行面）   │    │  （思考面）   │
│              │    │              │
│  线性·聚焦   │    │  网状·发散   │
│  此刻该做什么 │    │  全局长什么样 │
│  单手10秒    │    │  双手慢探索   │
└──────────────┘    └──────────────┘
         │                  │
         └──── 长按穿越 ─────┘
```

### 三个手势，一个界面

| 手势 | 效果 |
|------|------|
| **上滑**（从底部） | 呼出行动面板——线性、聚焦、为执行而生 |
| **下拉**（从顶部） | 进入认知地图——网状、发散、为思考而生 |
| **长按穿越** | 在行动卡片上长按→地图中定位；在地图节点上长按→行动面板中定位 |

录音入口始终在中间，两面都能触达。

## 执行面：行动面板

### 触发方式

任何状态下拇指从底部上滑，半屏毛玻璃面板覆盖在当前视图上方。

### 面板结构（三个区域）

```
┌─────────────────────────────┐
│                             │
│ ┌─ 此刻（Now）────────────┐ │
│ │                         │ │
│ │ 供应链评估               │ │
│ │ → 打给张总确认铝材报价    │ │
│ │   上次通话 3/15          │ │
│ │   他答应本周回复     📞  │ │
│ │                         │ │
│ │   ← 跳过    完成 →      │ │
│ └─────────────────────────┘ │
│                             │
│ ── 今日线 ──────────────── │
│ ● 打给张总（供应链评估）     │
│ ○ 14:00 审阅成本对比        │
│ ○ 明天前 回复老王邮件        │
│ ◇ 记录本周供应链进展         │
│                             │
│  ● ○ ○ ○                   │
│  供应链 产品 团队 个人        │
└─────────────────────────────┘
```

#### 区域一：此刻卡片

系统认为你**此刻最应该做的一件事**。

- 一张卡片，一句话，一个可操作元素（📞 / 📝 / 💬）
- **左滑**：跳过，换下一件
- **右滑**：标记完成
- **长按**：穿越到地图——看到这个行动的认知链全貌

此刻卡片的内容从哪来：所有活跃目标的待执行 Action Strike，按 `紧迫度 × 重要度 × 可执行性` 排序取 top-1。

#### 区域二：今日线

今天还需要做的事，**按最佳执行顺序排列**。最多 4-5 项，超出折叠。

| 符号 | 含义 |
|------|------|
| ● | 下一个要做的 |
| ○ | 后面的（有时间约束） |
| ◇ | 不限时间但相关 |

每项一行字。点击展开 → 从 Strike + Bond 中调取上下文。默认不展开。

#### 区域三：目标呼吸指示器

一排小圆点，每个代表一个活跃目标。当前目标高亮。左右滑切换目标。

### 模糊意图场景

用户不知道该做什么时上滑。面板顶部出现：

```
你有多少时间？
[5分钟] [15分钟] [1小时]
```

选择后，此刻卡片和今日线根据时间窗口重新筛选。推荐逻辑考虑：
- **时间匹配**：5 分钟不推荐 2 小时的深度工作
- **能量匹配**：下午 3 点推低认知负荷的执行任务
- **动量利用**：刚完成供应链任务→推同领域下一个（切换成本最低）

### 行动面板数据来源

面板是**地图的线性投影**。后端预计算（每日早晨 + 每次新 Strike 入库时）：

1. 找活跃目标（Intend 类 Strike，salience 最高）
2. 沿认知链找待执行 Action Strike
3. 按 `紧迫度 × 重要度 × 可执行性` 排序
4. 裁剪为 top 4-5 项

用户上滑时看到的是已算好的结果，不等大模型。

## 思考面：认知地图

### 触发方式

从顶部下拉，或从 Level -1 上滑（两种入口）。

### 缩放层级

```
Level -1: 纯净入口（倾倒模式）——默认态
    ↕ 上滑 / 捏合缩小
Level 0:  人生全景（3-5 个核心领域）
    ↕ 捏合缩放
Level 1:  领域展开（Cluster 卡片墙）
    ↕ 捏合缩放
Level 2:  Cluster 展开（认知时间线 + 模式 + 对立观点 + 目标状态）
    ↕ 捏合缩放
Level 3:  单条 Strike 详情（溯源到原始记录）
```

### Level -1: 纯净入口（默认态）

```
┌─────────────────────────────┐
│                             │
│                             │
│         [ 时间 ]            │
│                             │
│    最近一条关联提示（可选）   │
│                             │
│          🎙                  │
│      按住说话 / 点击输入      │
│                             │
│    ↑ 上滑呼出行动面板        │
│    ↓ 下拉进入认知地图        │
└─────────────────────────────┘
```

- 近乎空白。无地图、无 cluster、无洞察
- 录完音后底部浮出关联提示，5 秒后消失
- 不干扰思考

### Level 0: 人生全景

```
┌─────────────────────────────┐
│        我的认知世界           │
│                             │
│   ┌────────┐  ┌────────┐   │
│   │ 事业    │  │ 家庭    │   │
│   │ 28条    │  │ 12条    │   │
│   │ ●●●○   │  │ ●●○○   │   │
│   └────────┘  └────────┘   │
│                             │
│   ┌────────┐  ┌────────┐   │
│   │ 成长    │  │ 健康    │   │
│   │ 15条    │  │ 5条     │   │
│   │ ●●○○   │  │ ●○○○   │   │
│   └────────┘  └────────┘   │
│                             │
│         🎙 录音始终可用       │
└─────────────────────────────┘
```

- 圆点 = 近期记录密度（活跃度），不是评分
- 无判断性标签

### Level 1: 领域展开（Cluster 卡片墙）

```
┌─────────────────────────────┐
│ ← 事业                 28条  │
│                             │
│ ┌─────────────────────────┐ │
│ │ 供应链管理               │ │
│ │ 12条记录 · 存在对立观点   │ │
│ │ 最近记录 3/19            │ │
│ └─────────────────────────┘ │
│                             │
│ ┌─────────────────────────┐ │
│ │ 产品开发                 │ │
│ │ 8条记录 · 近两周活跃      │ │
│ └─────────────────────────┘ │
│                             │
│ ┌─────────────────────────┐ │
│ │ 团队建设                 │ │
│ │ 3条记录 · 最后记录14天前  │ │
│ └─────────────────────────┘ │
│                             │
│ 💬 长按 Cluster 可拖线连接   │
└─────────────────────────────┘
```

- **纯数据描述**：条数 + 最后记录时间 + 客观特征（存在对立观点）
- AI 注释层：长按才显示 AI 解读
- 可拖线建立手动连接

### Level 2: Cluster 展开

```
┌─────────────────────────────┐
│ ← 供应链管理           12条  │
│                             │
│ ┌─ 认知模式（AI 注释）───┐   │
│ │ "你面对成本问题时倾向于  │   │
│ │  更换而不是优化"        │   │
│ │  这准确吗？ [是] [否]  │   │
│ └─────────────────────────┘ │
│                             │
│ ┌─ 对立观点 ──────────────┐ │
│ │ ⚖️ "应该换" vs "风险大" │ │
│ │ [帮我想想这个问题]       │ │
│ └─────────────────────────┘ │
│                             │
│ ┌─ 目标状态 ──────────────┐ │
│ │ 🎯 建立供应商评估体系    │ │
│ │ ┌──┬──┬──┬──┐          │ │
│ │ │方向│资源│路径│驱动│     │ │
│ │ │ ██ │ ▓░ │ ░░ │ ██ │     │ │
│ │ └──┴──┴──┴──┘          │ │
│ │ 方向和驱动力较清晰，      │ │
│ │ 资源和路径还在积累中      │ │
│ └─────────────────────────┘ │
│                             │
│ ── 认知时间线 ─────────────  │
│ 3/19 👁 原材料涨了15%       │
│      ⚖️ 我认为应该换供应商   │
│      🎯 让小李做成本对比     │
│ 3/15 👁 供应商交期又延迟了   │
│ 3/12 💡 供应链是系统性短板   │
└─────────────────────────────┘
```

- 认知模式用**疑问语气** + 确认入口
- 目标四要素用**光谱**不用红绿灯
- 对立观点提供"帮我想想"入口 → 决策工作台

### Level 3: Strike 详情

溯源到原始录音。可编辑 nucleus/polarity，可重新分类。

## 穿越交互

面板（线性执行面）和地图（网状思考面）之间的无缝切换。

| 起点 | 手势 | 终点 |
|------|------|------|
| 行动面板的卡片 | 长按 | 地图中该 Strike 位置高亮，展示认知链 |
| 地图中的 Strike | 长按 | 行动面板中该 Strike 对应的行动卡片 |
| 行动面板 | 下滑/捏合缩小 | 地图全景 |
| 地图 | 上滑 | 行动面板 |

**面板是地图的线性投影，地图是面板的立体展开。翻手掌般自然——手心是行动面，手背是认知面。**

## 录入后关联（两阶段）

**第一阶段（即时）**：录完音 → Level -1 底部动画确认"已收录"

**第二阶段（异步，Digest 完成后）**：浮出关联提示：
> "关联到「供应链管理」，本周第 4 次提到。"

5 秒消失，点击跳转。

如果在地图视图中录音：
- 第一阶段：Strike 出现在中心，脉冲
- 第二阶段：缓缓漂流到归属 Cluster，经过的关联 Cluster 发光
- 可拖拽修正归属

## 推送逻辑（正确的时间提醒正确的事）

### 行动面板自动刷新

每日早晨 + 每次新 Strike 入库 → 后端重算行动面板内容 → 推送到前端缓存。

### 三层推送

| 层 | 时机 | 内容 | 语气 |
|----|------|------|------|
| 实时 | 录完音后 3 秒 | 关联提示 | 陈述 |
| 日常 | 晨间/到期时 | 今日行动 + deadline | 陈述 |
| 洞察 | 矛盾/模式发现时 | 认知洞察 | 疑问 + 确认 |

### 行为偏差（置信度驱动）

确定信号直接展示：
> "最近 4 周提到跑步 2 次，提到加班 11 次"

推断用疑问语气：
> "最近很少提到运动——是计划有变，还是只是没录到？"
> [计划有变] [没录到] [在坚持，只是没说]

## 冷启动

### 第一次打开

播种对话（非空地图）：

```
聊聊你最近在忙什么？
今年最想实现的一件事？
有什么一直想做但没开始的？

🎙 说说看  ⌨️ 打字  [跳过]
```

回答转化为初始 Strike → 3-5 个种子节点。

### 前 10 条记录

- 每条都给关联反馈
- 第 3 条：建立第一个 link
- 第 5 条：第一次微聚类
- 第 10 条：完整 Digest → 地图有实质内容

### 种子→发芽隐喻

1-3 条：种子散落
4-7 条：发芽，出现根系连接
8-10 条：幼苗，形成可辨认区域
10+：正常 Cluster 卡片墙

## 用户手动连接

### Cluster 级别（Level 1）

长按 Cluster → 拖线到另一个 Cluster → 创建手动 link
手绘风格虚线（vs 系统的实线）
系统沿这条线深挖隐性关联

### Strike 级别（Level 2）

长按 Strike → 拖向另一个 Strike → 创建 Bond
弹出 type 选择：因为 / 矛盾 / 支持 / 后来变成了

手动连接权重 > 系统自动连接。

## 后端支持（新增）

### 行动面板计算

```typescript
// 新增：gateway/src/cognitive/action-panel.ts
export async function computeActionPanel(userId: string): Promise<ActionPanel>

interface ActionPanel {
  now: ActionCard;              // 此刻最应做的一件事
  today: ActionItem[];          // 今日线（4-5项）
  goals: GoalIndicator[];       // 目标呼吸指示器
}

interface ActionCard {
  strikeId: string;
  goalName: string;
  action: string;              // 一句话行动描述
  context?: string;            // 上下文（上次通话日期等）
  actionType: 'call' | 'write' | 'review' | 'think' | 'record';
  targetPerson?: string;
}
```

触发：每日早晨 cron + 每次 Digest 完成后。

### 关联提示计算

Digest 完成后，生成一句关联提示推送到前端：
- 最相关的 Cluster 名称
- 本周提及次数
- 最强的跨记录 Bond 描述
