# 念念有路（V2Note）— 产品需求文档 PRD

> 版本：v2.0 | 反向工程自代码库 | 2026-03-29

---

## 一、产品概述

### 1.1 产品定位

**念念有路**是一款 AI 驱动的个人认知操作系统，帮助用户管理从"想法"到"行动"的完整链路。

用户不需要思考如何分类、归档、整理——只需把脑海中的一切（碎碎念、工作笔记、灵感、吐槽、读书感想、账单、行程……）倾倒进来，系统会自动提取认知、发现主题、识别矛盾、生成待办、追踪目标，并在合适的时间给出回顾与建议。

### 1.2 核心理念

| 原则 | 含义 |
|------|------|
| **混沌输入** | 用户可以把任何东西丢进来，不需要预先分类 |
| **结构涌现** | 主题和目标从内容密度中自然生长，不做手动分类 |
| **AI 沉默为主** | AI 只在每日回顾中主动发言，不在其他场景打扰用户 |
| **感受不分析** | 情感类记录只记录不做逻辑分析，尊重用户的感受自主权 |
| **素材降权** | 外部素材（文章、链接等）权重极低，不参与主题涌现 |

### 1.3 目标用户

知识工作者、创业者、自由职业者——日常有大量碎片化想法和待办，但缺少系统性工具将思考转化为行动的人群。

### 1.4 技术栈

- 前端：Next.js 16 + TypeScript + Capacitor 8（Android/iOS）+ Electron（桌面）
- 后端：Node.js Gateway（WebSocket + REST）
- 数据库：Supabase（PostgreSQL + pgvector 向量搜索）
- AI：DashScope（阿里通义）语音识别 + 大模型推理
- UI：shadcn/ui + Tailwind CSS + Radix UI

## 二、核心数据模型

### 2.1 三层认知模型

```
Strike（触动）→ Bond（关系）→ Cluster（聚类涌现）
```

#### Strike（认知触动）

系统最小的语义单元——用户一次表达中的一个独立认知。

| 字段 | 说明 |
|------|------|
| nucleus | 语义核心（如"铝价涨了15%"） |
| polarity | 认知极性：perceive（感知）/ judge（判断）/ realize（领悟）/ intend（意图）/ feel（感受） |
| salience | 重要性权重 0~1，随时间衰减；素材类强制降为 0.1~0.2 |
| confidence | AI 对极性判断的置信度 0~1 |
| field | 上下文（时间、生活阶段、空间、精力、情绪、社交场景） |

#### Bond（关系）

连接 Strike 之间的有向关系。

- 类型：causal（因果）、contradiction（矛盾）、resonance（共鸣）、evolution（演进）、supports（支持）、context_of（背景）、elaborates（展开）、triggers（触发）、resolves（解决）、depends_on（依赖）等
- 强度 0~1，长期不被引用则衰减，反复出现则增强

#### Cluster（聚类）

相关 Strike 自然聚合形成的主题。

- 三级层次：L1（主题）→ L2（元主题）→ L3（人生领域）
- 可与 Goal 关联（一对多）
- AI 自动创建，无需用户手动归类

### 2.2 统一任务模型

Goal 和 Todo 统一为同一张表，通过 `level` 区分：

| level | 含义 | 示例 |
|-------|------|------|
| 0 | 原子待办 | "给张总打电话确认报价" |
| 1 | 单次目标 | "完成供应链优化方案" |
| 2 | 多次项目 | "Q2 产品发布" |
| 3 | 人生领域 | "职业发展" |

## 三、功能模块

### 3.1 输入层 — 混沌入口

#### 3.1.1 语音录入

**交互流程：**

1. 用户点击底部悬浮按钮（FAB），进入录音模式
2. 按住录音，松手结束（支持锁定长按模式）
3. 实时 ASR 转写，流式显示文字
4. 转写完成 → AI 自动分类意图 → 执行相应动作

**技术细节：**
- 32 段音量可视化波形
- 支持暂停/继续录音
- 实时 ASR 通过 Python 子进程调用 DashScope
- 录音数据以 PCM 格式通过 WebSocket 流式传输

#### 3.1.2 文字输入

- 移动端：底部输入栏，点击弹出文字编辑面板
- PC 端：富文本编辑器，支持 Markdown、`/` 命令面板、`@` 引用主题/目标、`#` 标签
- 支持拖拽上传图片/文件/URL

#### 3.1.3 意图分类（AI 自动判断）

| 模式 | 触发条件 | 执行 |
|------|---------|------|
| **纯记录** | 叙述性内容（"今天开会讨论了铝材行情"） | 保存日记 + 提取 Strike |
| **纯行动** | 指令性内容（"把张总那个改到明天3点"） | 直接执行工具操作 |
| **混合** | 叙述 + 指令（"开会讨论涨价，提醒我明天问报价"） | 保存日记 + 提取 Strike + 创建待办 |

### 3.2 认知引擎 — 双层架构

#### 3.2.1 Tier-1：实时 Strike 提取

**触发：** 每条记录处理后立即执行

**单次 AI 调用完成：**
- 将记录分解为 5~20 个 Strike
- 判断每个 Strike 的极性（perceive/judge/realize/intend/feel）
- 创建 Strike 之间的内部 Bond
- 自动提取标签（领域、人名、关键词）
- intend 类 Strike 自动投射为待办/目标
- 异步更新用户画像和 AI 人格

**性能指标：** ≤5 秒/条记录

#### 3.2.2 Tier-2：批量分析

**触发条件：**
- 累计 ≥5 个新 Strike
- 每日凌晨 3:00
- 用户手动触发

**单次 AI 调用完成：**
- 将新 Strike 分配到已有 Cluster
- 发现新主题 → 创建 L1 Cluster
- 检测矛盾（contradiction）
- 发现认知模式（pattern）
- 建议合并相似 Cluster
- 推荐新目标

**性能指标：** ≤20 秒（50~100 个新 Strike）

#### 3.2.3 L2 涌现

- 当 3+ 个 L1 Cluster 高度相关时，自动聚合为 L2 元主题
- 每周执行或自动触发
- 基于图论的连通分量检测 + AI 判断主题相似性

### 3.3 每日认知循环

#### 3.3.1 晨间简报（7:30 推送）

| 模块 | 内容 |
|------|------|
| 问候 | 个性化问候，使用用户名称 |
| 今日焦点 | 编号优先事项列表 |
| 目标进展 | 活跃目标状态条 |
| 遗留待办 | 昨日未完成项（支持直接标记完成） |
| 中转提醒 | 需要传达给他人的事项 |
| AI 建议 | 基于认知分析的行动建议 |

#### 3.3.2 晚间回顾（20:00 推送）

| 模块 | 内容 |
|------|------|
| 极性光谱 | 今日 vs 7日均值对比 |
| 高价值领悟 | 今日最有价值的 realize Strike |
| 完成率 | 待办完成统计 |
| 情绪弧线 | feel Strike 时间轴 |
| 反思问题 | 1~2 个基于实际 Strike 的追问 |

#### 3.3.3 日终维护（凌晨 3:00 自动执行）

1. **批量分析** — Tier-2 认知引擎
2. **衰减维护** — Bond 强度和 Strike 显著性随时间衰减
3. **认知报告** — 生成每日认知摘要，写入 AI 日记

### 3.4 待办系统

#### 3.4.1 智能提取

从语音/文字中自动提取：

| 字段 | 示例 |
|------|------|
| 时间 | "明天下午3点" → scheduled_start |
| 人物 | "张总" → person |
| 优先级 | "挺急的" → high |
| 领域 | work / life / social / learning / health |
| 影响力 | 1~10（AI 评估） |
| AI 可执行 | 是否可以由 AI 辅助完成 |
| 预计时长 | AI 估算分钟数 |

#### 3.4.2 行动面板（移动端核心交互）

- **NowCard**：当前聚焦的目标/任务，大号完成按钮
- **目标轮播**：左右滑动切换不同目标
- **今日待办列表**：
  - 按目标/领域分组
  - 复选框切换完成状态
  - 影响力可视化（1-3 隐藏，4-6 数字，7-8 火焰，9-10 脉冲火焰）
  - AI 可执行项显示紫色星光图标
- **待办详情底板**：
  - 日期/时间选择器
  - 时长估算
  - AI 执行计划（分步骤列表 + "让AI帮忙"按钮）
  - 来源日记引用

#### 3.4.3 主动提醒

| 场景 | 行为 |
|------|------|
| 高影响待办未排期 | 推送提醒"你有 N 项待办还没安排时间" |
| 连续跳过 3 次 | 预警"这个任务被跳过多次，需要重新评估吗？" |
| 7 天后跟进 | "报价确认一周了，结果怎样？" |
| AI 可执行 + 高影响 | 特殊提醒消息 |

#### 3.4.4 领域配色

| 领域 | 颜色 |
|------|------|
| 工作 work | 蓝色 |
| 生活 life | 绿色 |
| 社交 social | 琥珀色 |
| 学习 learning | 青色 |
| 健康 health | 玫红色 |

### 3.5 目标系统

#### 3.5.1 目标生命周期

```
suggested（AI 建议）
    → active（用户确认）
    → progressing（首个待办完成）
    → blocked（待办连续跳过 3+ 次）
    → completed（全部待办完成 + 用户确认）
    → archived（归档）
```

#### 3.5.2 目标健康度（四维评分）

| 维度 | 计算方式 |
|------|---------|
| **方向** Direction | Cluster 中 intend Strike 占比 |
| **资源** Resource | perceive Strike 数量（信息充分度） |
| **路径** Path | 关联待办完成率 |
| **驱动力** Drive | feel/judge Strike 存在度（情感动力） |

#### 3.5.3 目标自动关联

- 创建目标时：扫描 Cluster 内相关 Strike
- 新记录归类到目标 Cluster → 自动关联
- 待办通过关键词匹配自动链接到目标

### 3.6 对话 & 智能体系统

#### 3.6.1 对话模式

| 模式 | 场景 | 示例 |
|------|------|------|
| review | 日期范围回顾 | "回顾一下这周" |
| command | 任务指令 | "帮我创建一个目标" |
| insight | 深度分析 | 苏格拉底追问、芒格审查 |
| decision | 决策支持 | "我该不该转行做AI？" |

#### 3.6.2 工具系统（AI 可调用）

| 工具 | 功能 | 确认级别 |
|------|------|---------|
| search | 搜索 Strike/待办 | 静默 |
| fetch_url | 获取网页内容 | 静默 |
| create_record | 创建日记 | 流式 |
| create_todo | 创建待办 | 流式 |
| update_todo | 修改待办 | 流式 |
| delete_todo | 删除待办 | 需确认 |
| create_goal | 创建目标 | 需确认 |
| create_project | 创建项目 | 需确认 |
| web_search | 网络搜索 | 静默 |

#### 3.6.3 Plan 机制（多步骤复杂任务）

**场景示例：** "帮我拆解 Q2 产品发布"

**流程：**
1. AI 识别为复杂任务 → 生成结构化计划
2. 计划持久化到数据库
3. 用户在聊天中看到 PlanCard（执行 / 修改 / 取消）
4. 确认后 → 批量执行每个步骤
5. 步骤失败 → 提示用户保留/回滚

**状态流转：** drafting → awaiting_confirm → executing → paused → done

### 3.7 意图理解系统

五种意图类型的自动路由：

| 意图 | 示例 | 处理 |
|------|------|------|
| task | "买牛奶" | 直接创建待办 |
| wish | "想学弹吉他" | 存为 pending_intent，等待用户确认 |
| goal | "Q2 之前拿到 PMP 证书" | 创建 pending_intent + 建议目标 |
| complaint | "又加班，烦死了" | 写入 Soul 系统（AI 人格吸收） |
| reflection | "原来项目管理的关键在于沟通" | 写入 Memory 系统（长期记忆） |

### 3.8 记忆架构

#### 三层分离：

| 层 | 存储 | 更新频率 | 用途 |
|----|------|---------|------|
| **Soul**（人格） | AI 的性格、语气、禁忌、交互偏好 | 低频（用户驱动） | 塑造 AI 表达风格 |
| **User Profile**（画像） | 用户的职业、习惯、作息、偏好 | 中频（每次交互提取） | 个性化建议 |
| **Memory**（记忆） | 长期洞察：事件、决策、观察、目标 | 高频（交互驱动） | 上下文理解 |

记忆更新采用 Mem0 两阶段机制：提取 → 比对 → 决策（ADD/UPDATE/DELETE/NONE）

### 3.9 搜索系统

**混合检索策略：**

- **语义搜索**：embedding 余弦相似度（Top-5）
- **结构化搜索**：标签交集、人名正则、时间窗口（±7天）、极性匹配
- **综合排名**：`similarity × 0.6 + structuredHits × 0.4`

**搜索结果类型：** Strike（日记）、Todo（待办）、Goal（目标）、People（人物）

### 3.10 冷启动引导

5 步问卷快速建立用户画像：

| 步骤 | 问题 | 产出 |
|------|------|------|
| Q1 | 怎么称呼你？ | UserProfile.name |
| Q2 | 你的人生阶段？ | 种子维度（L3 Cluster） |
| Q3 | 当前最关注什么？ | 首个目标建议 |
| Q4 | 最大的痛点？ | 功能优先级配置 |
| Q5 | 最佳回顾时间？ | 推送时间设置 |

**结果：** 4 条种子日记 → Tier-1 提取 10~20 个初始 Strike → 主题初始化 → 目标建议 → 可以开始使用

### 3.11 通知系统

| 类型 | 图标 | 颜色 |
|------|------|------|
| 晨间简报 | 🌅 | 晨曦色 |
| 晚间回顾 | 🌙 | 天空色 |
| 待办提醒 | 🎯 | 鹿棕色 |
| 中转提醒 | ☎️ | 森林色 |
| 认知预警 | ⚡ | 枫叶色 |

### 3.12 领域词汇系统

- 冷启动：用户选择 3~5 个初始领域
- 自动扩充：从 Strike 中提取新领域词汇
- 同义词：AI 生成（如 "编程" ↔ "开发"）

- 用途：待办领域分类、Cluster 极性分析、维度导航

## 五、核心用户流程

### 流程 1：语音记录 → 自动待办

```
用户点击 FAB → 说话："明天下午3点找张总确认报价，挺急的"
    ↓
ASR 实时转写 → AI 分类为"混合"模式
    ↓
日记保存 + Strike 提取（intend 类）
    ↓
待办自动投射：scheduled_start=明天15:00, person=张总, priority=high
    ↓
AI 回复："已加到明天日程，下午3点提醒你"
```

### 流程 2：矛盾发现 → 引导反思

```
第 1 天：用户记录"我应该转行做AI"
第 7 天：用户记录"AI太卷了，还是稳定工作重要"
    ↓
Tier-2 分析：检测到 contradiction Bond
    ↓
晚间回顾注入："我发现你最近对职业有两种想法，这可能需要深入思考"
    ↓
用户点击"深入了解" → 进入苏格拉底追问模式
```

### 流程 3：目标涌现 → 任务拆解

```
用户 2 周内录入 5+ 条关于"供应链"的记录
    ↓
Tier-2：检测到 intend 密度 ≥35% → 建议目标"供应链优化"
    ↓
用户在聊天中说"帮我拆解一下"
    ↓
AI 生成 Plan：5 步行动计划（调研、对比、评估、谈判、实施）
    ↓
用户审核 → 修改步骤2 → 确认执行
    ↓
批量创建 5 个待办 + 3 个子目标，自动链接到 Cluster
```

### 流程 4：日常闭环

```
07:30  晨间简报：3 个高影响待办未排期，昨日遗留 2 项
  ↓
08:00  用户安排待办时间
  ↓
14:00  AI 提醒跳过的高影响任务："需要帮助吗？"
  ↓
18:00  用户语音记录："今天把供应商谈妥了"
  ↓
```

## 八、非功能性需求

### 8.1 性能

| 指标 | 目标 |

### 8.2 离线支持

- 前端离线时显示 OfflineBanner
- 本地缓存待办/日记列表
- 录音本地暂存，恢复网络后上传
- 优雅降级：Redis 不可用 → 回退到 setInterval 调度

### 8.3 安全

- JWT + Refresh Token 认证
- 设备关联（device_id + user_id 双维度）
- 高风险操作需要确认（删除、创建目标）
- 30 秒确认超时
