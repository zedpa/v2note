# 意图分类

从用户的语音或文字记录中识别每个有意义的语句，分类为以下类型：

## 分类规则

| type | 判断标准 | 示例 |
|------|---------|------|
| `task` | 主体+动作+客体，三要素齐全，可立即执行 | "明天给张总打电话" |
| `wish` | "我想/想要/希望" + 无明确下一步 | "想学弹吉他" |
| `goal` | 较大目标，有时间维度或可衡量 | "今年把身体搞好" |
| `complaint` | 负面情绪、抱怨、不满 | "最近太累了" |
| `reflection` | 自我反思、领悟、总结 | "我发现我总是拖延" |

## 判断优先级

1. 如果三要素（主体+动作+客体）齐全且可立即执行 → `task`
2. 如果是"我想/想要/希望/打算" + 无具体行动步骤 → `wish`
3. 如果涉及长期规划、年度/季度目标 → `goal`
4. 如果表达不满、疲惫、压力 → `complaint`
5. 如果是对自己行为/习惯的反思 → `reflection`
6. 不属于以上任何类型的普通叙述 → 不提取

## task 类型的提取规则（继承自 todo-extract）

只有同时满足三要素的句子才归类为 task：
- **主体**: 谁来做——人名、角色或"我"
- **动作**: 做什么——明确的动词/行为
- **客体**: 对什么——动作的目标或内容

如果主体是"我"，省略主体，直接以动词开头。

## 输出格式

在 `intents` 数组中返回所有识别到的意图：

```json
{
  "intents": [
    { "type": "task", "text": "明天给张总打电话" },
    { "type": "wish", "text": "学弹吉他", "context": "想培养新爱好" },
    { "type": "goal", "text": "今年把身体搞好", "context": "最近体检指标不好" }
  ]
}
```

每个意图对象包含：
- `type`: 意图类型（必填）
- `text`: 精简的意图描述（必填）
- `context`: 上下文说明，为什么用户提到这个（可选，wish/goal 建议填写）

## 示例

| 原文 | type | text | context |
|------|------|------|---------|
| "我明天要给张总打电话" | task | "明天给张总打电话" | - |
| "想学弹吉他" | wish | "学弹吉他" | "想培养新爱好" |
| "今年一定要把身体搞好" | goal | "今年把身体搞好" | - |
| "最近加班太多了，好累" | complaint | "加班太多，很累" | - |
| "我发现我总是把重要的事拖到最后" | reflection | "总是把重要的事拖到最后" | "拖延习惯" |
| "今天天气真好" | - | - | 不提取 |
