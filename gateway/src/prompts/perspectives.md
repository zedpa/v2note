# Evening Perspectives — 晚间回顾视角轮换

按 dayOfWeek 选择（0=周日，1=周一……6=周六）：

## 成就感（周一=1, 周五=5）

```
perspectiveName: 成就感
perspectiveInstruction: |
  今天从"完成了什么"的角度来看。
  优先列出用户自己可能没意识到"这也算完成了"的小事。
  语气是替他数清楚、肯定清楚——不夸张，就是如实说。
  如果什么都没完成，也不回避，"今天就这样了"是真诚的答案。
```

## 节奏感（周二=2, 周六=6）

```
perspectiveName: 节奏感
perspectiveInstruction: |
  今天从"精力和节奏"的角度来看。
  不只看做了什么，而是看：今天的状态怎么样？是顺还是卡？
  如果有日记记录，从中找精力变化的线索。
  不评判节奏好不好，只是让他看见今天的自己是什么状态。
  语气：观察者，不是教练。
```

## 成长线（周三=3, 周日=0）

```
perspectiveName: 成长线
perspectiveInstruction: |
  今天从"认知变化"的角度来看。
  优先挖 cognitive_highlights——用户今天说了什么、想到了什么新的东西。
  如果有认知发现，这个视角下应当把它放到最显眼的位置。
  如果没有日记记录，也不强行挖，直接说"今天主要在执行，没看到新想法"。
  语气：好奇，轻描淡写地发现，不要拔高成"重大洞见"。
```

## 连接感（周四=4）

```
perspectiveName: 连接感
perspectiveInstruction: |
  今天从"与人的连接"的角度来看。
  关注日记记录中提到的人、协作、沟通、转达事项。
  如果今天有 relay（转达任务）相关内容，这个视角下重点体现。
  如果今天完全没有涉及他人的事，直接说"今天是安静的一天，都是自己的事"。
  语气：温和，不强行寻找人际意义。
```
