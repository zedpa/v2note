# 阅读器

> 状态：🟡 待开发 | 优先级：Phase 5 | 预计：3-4 天

## 概述
长日记、每日回顾报告、参谋分析——都需要舒适的阅读体验。当前长内容在卡片中密排，没有专门的阅读模式。

## 场景

### 场景 1: 长内容进入阅读模式
```
假设 (Given)  用户展开一条超过 500 字的日记
当   (When)   内容超过阈值
那么 (Then)   卡片底部出现"阅读全文 →"按钮
并且 (And)    点击进入全屏阅读模式
```

### 场景 2: 阅读器排版
```
假设 (Given)  进入阅读模式
那么 (Then)   居中排版 max-width 640px
并且 (And)    字体 Noto Serif SC 18px / 行高 1.8
并且 (And)    顶部：日期 + 来源图标 + 所属 Cluster 标签
并且 (And)    底部：相关记录列表 + 关联目标
并且 (And)    Esc 或左上角返回
```

### 场景 3: 选中文字交互
```
假设 (Given)  用户在阅读器中选中一段文字
当   (When)   松开选择
那么 (Then)   浮出工具栏：高亮 | 批注 | 问路路 | 建立链接
并且 (And)    工具栏位于选中文字上方
并且 (And)    点击外部区域自动消失
```

### 场景 4: 阅读中问路路
```
假设 (Given)  用户选中一段文字后点击"问路路"
当   (When)   参谋对话打开
那么 (Then)   上下文包含：当前日记全文 + 选中段落高亮标记
并且 (And)    路路回复围绕选中内容展开
```

### 场景 5: 每日回顾的阅读体验
```
假设 (Given)  路路的晚间报告 800 字
当   (When)   用户打开每日回顾
那么 (Then)   以阅读友好排版呈现（非弹窗密排）
并且 (And)    行动建议可直接勾选
并且 (And)    洞察引用可点击跳转原始日记
并且 (And)    反思引导可直接回复（回复变成新日记）
```

### 场景 6: 素材阅读器
```
假设 (Given)  用户拖入 PDF 并点击"查看全文"
当   (When)   阅读器打开
那么 (Then)   渲染提取的文本内容
并且 (And)    侧栏显示路路摘要 + 关键提取
并且 (And)    用户选中段落可"添加想法"（创建 think 日记，关联到素材）
```

## 验收标准
长日记有舒适的全屏阅读体验；选中文字能直接问路路；每日回顾以阅读友好排版呈现。
