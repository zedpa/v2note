# v2note 推广执行手册

> 配合 GTM-PLAN.md 使用。本文档是具体的执行 checklist。

---

## Phase 0：自用验证（14 天）

### Day 1-3：基础准备

- [ ] **录屏环境搭建**
  - 安卓手机安装录屏工具（推荐 AZ Screen Recorder，支持内录声音）
  - 设置 9:16 竖屏录制（1080x1920）
  - 准备一个干净的测试账号（清空历史数据）

- [ ] **内测申请页制作**
  - 使用腾讯问卷/金数据创建表单
  - 字段：姓名、职业、日均开会次数、用过哪些待办工具、为什么放弃、手机型号
  - 页面只有：一句话 + 30 秒视频 + 表单
  - 部署到 `v2note.com/beta`（或用问卷链接代替）

- [ ] **社交媒体账号注册**
  - 即刻：头像用产品 logo，简介"做一个说了就行的 AI 助手"
  - 小红书：同上
  - Twitter/X：英文账号，"Building an AI voice execution engine"

### Day 4-14：全天候自用 + 素材积累

**每天必做**：

| 时间 | 行动 | 产出 |
|------|------|------|
| 早上 | 打开晨间简报，截图 | 简报截图 ×1 |
| 全天 | 所有事情用语音记录（不用打字） | 使用时长数据 |
| 遇到好效果时 | 立即录屏 15 秒 | 短视频素材 ×2-3/天 |
| 晚上 | 记录今天的"哇"时刻和"烦"时刻 | 体验日志 ×1 |

**14 天结束时应有**：

- [ ] 30+ 个 15 秒短视频片段
- [ ] 14 张晨间简报截图
- [ ] 一份完整的体验日志（记录痛点和惊喜）
- [ ] 修复了至少 5 个自己发现的 bug
- [ ] 能流利地用 30 秒演示产品核心价值

---

### 核心演示视频制作（Day 12-14）

#### 脚本

```
[场景] 从素材中选最好的 3 个片段拼接

片段 A（6s）：你对着手机说一句话
  - 选择一句包含多件事的自然语句
  - 要真实，不要念稿

片段 B（8s）：屏幕录制 - 待办自动分类
  - 录制从语音结束到待办列表出现的完整过程
  - 确保分类和影响力评分清晰可见

片段 C（6s）：屏幕录制 - 晨间简报 + AI 提醒
  - 展示简报界面
  - 展示"让AI帮我"按钮

前后各加 3 秒字幕卡
```

#### 技术规格

| 项 | 要求 |
|----|------|
| 画幅 | 9:16 竖屏 |
| 分辨率 | 1080×1920 |
| 时长 | 28-32 秒 |
| 字幕 | 大号黑体，白色，底部带半透明黑底 |
| 音乐 | 轻柔电子乐，推荐 Artlist/Epidemic Sound 免费曲库 |
| 剪辑工具 | 剪映（免费，支持自动字幕） |
| 转场 | 简单切换，不要花哨特效 |

## Phase 1：种子用户获取（Week 3-6）

### 即刻运营

#### 首发帖（W3 周一发布）

**发布时间**：周一早上 8:30（通勤高峰，即刻活跃时段）

**内容模板**：

```
我做了一个"说了就行"的 AI 助手，用了两周后回不去了

[详见 GTM-PLAN.md 素材 2]

现在开放 100 个内测名额。
如果你也是忙到没时间打字的人，留言或私信我。

#独立开发 #AI产品 #效率工具
```

**发布后动作**：
- 前 2 小时每条评论必回
- 主动 @ 3-5 个即刻上关注效率工具的大 V
- 如果数据好，下午 2 点转发到"产品猎人"/"效率工具"圈子

#### 后续内容节奏

| 帖子类型 | 频率 | 示例标题 |
|----------|------|----------|
| 使用场景 | 每周 1 | "开完会走出会议室的 10 秒钟，我把所有待办说完了" |
| 产品思考 | 每周 1 | "为什么我不做'笔记 app'，而做'语音执行引擎'" |
| 数据/进展 | 每周 1 | "内测第一周，用户说得最多的一句话是……" |

### 小红书运营

#### 封面设计规范

- 风格：深色背景 + 亮色手机 mockup
- 字体：思源黑体 Bold
- 标题放在图片上 1/3 处，不超过 10 个字
- 手机截图居中偏下

#### 内容类型轮换

| 类型 | 示例 | 制作难度 |
|------|------|----------|
| 对比图 | 之前 vs 之后（分屏） | 低 |
| 录屏演示 | 15 秒操作演示 | 中 |
| 场景故事 | "当妈以后怎么记事" | 中 |
| 功能揭秘 | "这个 AI 怎么知道哪件事更重要" | 高 |

### 用户反馈收集

#### 每周用户访谈（15 分钟电话）

**提问模板**：

1. 你这周用 v2note 最多的场景是什么？
2. 有没有哪个时刻觉得"哇，这个好用"？
3. 有没有哪个时刻觉得"这也太蠢了"？
4. 如果只能保留一个功能，你留哪个？
5. 你会推荐给谁？用什么话推荐？

**重点关注**：
- 问题 2 的答案 → 这是你的核心卖点
- 问题 4 的答案 → 这是你的 MVP 功能
- 问题 5 的答案 → 这是你的推广话术（用户的话比你的更好）

#### 反馈追踪表

| 日期 | 用户 | 反馈类型 | 内容 | 优先级 | 状态 |
|------|------|----------|------|--------|------|
| | | Bug/体验/建议 | | P0-P2 | 待处理/已修复 |

## Phase 2：口碑扩散（Week 7-12）

### 用户故事生产线

#### 步骤

1. 从 Phase 1 的访谈中筛选 5 个最打动人的故事
2. 征得用户同意后，协助他们写/录自己的体验
3. 格式：
   - 短版（即刻/Twitter）：200 字 + 1 张截图
   - 中版（小红书）：封面 + 4 张图 + 300 字
   - 长版（少数派/公众号）：2000 字深度体验

#### 用户故事模板

```
## [用户昵称]，[职业]

### 痛点
[一句话描述他之前的困境]

### 转折
[他开始用 v2note 的契机]

### 变化
[具体的、可感知的生活变化，最好有数据]

### 一句话
[他自己的原话推荐语]
```

### 产品驱动增长（PLG）机制

#### 内置传播触点

| 触点 | 机制 | 实现优先级 |
|------|------|------------|
| 晨间简报底部 | "由 v2note 生成" 水印 + 链接 | P0 |
| AI 复盘报告导出 | 生成精美长图，带 v2note logo | P1 |
| 周报生成 | "本周完成 N 件事" 分享卡片 | P1 |
| 邀请机制 | 邀请 1 个朋友 → 解锁 AI 执行功能 | P2 |

- [ ] 开始有用户问"怎么付费"

## 每日执行 checklist

### Phase 0 期间（每天 30 分钟）

- [ ] 全天用语音记录所有事情
- [ ] 录 2-3 个好效果的 15 秒视频
- [ ] 晚上写体验日志（3 句话：惊喜/困惑/想法）
- [ ] 修复当天发现的 bug（如有）

### Phase 1 期间（每天 1 小时）

- [ ] 回复所有社交媒体评论和私信
- [ ] 按日历发布内容
- [ ] 查看用户反馈，标记优先级
- [ ] 更新反馈追踪表

### Phase 2 期间（每天 1.5 小时）

- [ ] 社交媒体内容发布和互动
- [ ] 联系 1 个用户聊使用体验
- [ ] 推进 1 个用户故事的制作
- [ ] 分析增长数据，调整策略

---

*记住：推广的本质不是营销技巧，是产品本身好到用户忍不住告诉别人。先把产品做到那个程度。*
